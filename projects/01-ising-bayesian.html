Bayesian Inference on the 2D Ising Model

A pipeline that simulates the Ising spin lattice with Monte Carlo, infers the critical temperature and the critical exponent via MCMC in PyMC, and validates results against the exact Onsager (1944) solution.

Code on GitHub → See results
// Overview

About this project

This project combines three things rarely seen together in a junior ML portfolio: real statistical physics, applied Bayesian statistics and end-to-end scientific Python code. The 2D Ising model is ideal: it has a nontrivial phase transition and, at the same time, an exact analytical solution against which any numerical code can be validated.

A Numba-accelerated Metropolis simulator generates the M(T) curve of the system, and a probabilistic model in PyMC with physically motivated priors infers the critical parameters from that noisy curve. The comparison against Onsager is the ultimate test: if inference recovers `Tc ≈ 2.2692` and `β = 1/8` with reasonable uncertainty, the full pipeline is sound.

Everything runs in a single command: python main.py executes simulation, inference, and generates all final figures.

Core formulas

System energy:

E = −J ∑⟨i,j⟩ sᵢ · sⱼ

Metropolis acceptance rule:

P(flip) = min(1, e−ΔE/T)

Power law near Tc:

M(T) ≈ (1 − T/Tc)β

Bayes theorem applied:

P(Tc, β | data) ∝ P(data | Tc, β) · P(Tc, β)

Onsager's exact solution:

Tc = 2 / ln(1 + √2) ≈ 2.2692
β = 1/8 = 0.125
// Components

How it's built

Six pieces connected end-to-end. Each lives in an isolated module and exposes its own CLI.

1 · Metropolis simulator

From-scratch implementation of the Metropolis-Hastings algorithm to sample configurations from the canonical ensemble of the spin lattice. Acceptance rule uses the local energy change ΔE when flipping a spin, with probability min(1, exp(−ΔE/T)). Each sweep performs N² flip attempts — one per site on average. The critical kernel is accelerated with @numba.njit(cache=True) for a ~50× speedup over pure NumPy, with transparent fallback if Numba is unavailable.

2 · Temperature sweep

Sweep over 30 temperatures in T ∈ [1.5, 3.5], comfortably covering the theoretical Tc. For each T: 2000 thermalization sweeps + 3000 measurement sweeps, recording mean and standard deviation of ⟨|M|⟩. Output: a CSV with columns T, M_mean, M_std that serves as the dataset for inference.

3 · Probabilistic model

Bayesian model in PyMC with physically motivated priors: Tc ~ Normal(2.3, 0.3), β ~ Normal(0.12, 0.05), σ ~ HalfNormal(0.1). The prediction is the power law (1 − T/Tc)^β for T < Tc, and 0 in the disordered regime. Likelihood is Gaussian with σ also inferred.

4 · NUTS inference

Posterior sampling with NUTS: 4 chains, 2000 draws, 1000 tuning, target_accept=0.9. Diagnostics with ArviZ — R̂, ESS, trace plots — targeting R̂ < 1.01 and ESS > 400 for all parameters.

5 · Validation against Onsager

Exact values (Tc ≈ 2.2692, β = 1/8) are overlaid on the posterior distributions as reference lines. It's the strongest possible validation: numerical inference compared against an exact analytical result from 1944.

6 · Final figures

Three figures: (a) simulated M(T) curve with error bars and Onsager's Tc; (b) marginal posteriors of Tc and β with exact values overlaid; (c) trace plots for visual verification of MCMC convergence. Everything saved to results/.
