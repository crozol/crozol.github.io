Anomaly Detection API
Autoencoder + FastAPI + Docker

Un autoencoder LSTM aprende a reconstruir series multivariadas "sanas" de sensores (NASA Turbofan CMAPSS). Cuando recibe datos anómalos, el error de reconstrucción dispara la alerta. Todo expuesto como API REST con demo Streamlit y contenedorizado con Docker.

Código en GitHub → Demo (pendiente)
// Resumen

¿De qué se trata este proyecto?

Este es el proyecto pivote de ingeniería del portafolio. Mientras los otros proyectos demuestran profundidad matemática, este demuestra que puedo deployar algo real: dataset público, modelo entrenado, API documentada, contenedor reproducible, demo interactiva.

La técnica es detección de anomalías por reconstrucción: entrenas un autoencoder solo con series normales. Cuando recibe una serie anómala, falla al reconstruirla y el error pico sube. Simple, efectivo y ampliamente usado en industria.

Dataset: NASA Turbofan CMAPSS — series temporales de ~20 sensores de motores a reacción, con etiquetas de vida útil restante. Benchmark público, con métricas reproducibles.

Pipeline de inferencia

Entrenamiento (solo datos normales):

minθ E [‖x − AEθ(x)‖²]

Umbral de anomalía:

τ = percentil99(erroresval)

Decisión en producción:

anomaly = (‖x − AEθ(x)‖² > τ)
