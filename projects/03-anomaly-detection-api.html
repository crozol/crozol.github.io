Anomaly Detection API
Autoencoder + FastAPI + Docker

An LSTM autoencoder learns to reconstruct "healthy" multivariate sensor sequences (NASA Turbofan CMAPSS). When given anomalous data, reconstruction error spikes and triggers the alert. All served as a REST API with a Streamlit demo, containerized with Docker.

Code on GitHub → Demo (pending)
// Overview

About this project

This is the engineering pivot of the portfolio. While other projects showcase mathematical depth, this one shows real deployment: public dataset, trained model, documented API, reproducible container, interactive demo.

The technique is reconstruction-based anomaly detection: train an autoencoder only on normal sequences. When it receives anomalous data, it fails to reconstruct and the peak error rises. Simple, effective, and widely used in industry.

Dataset: NASA Turbofan CMAPSS — time series of ~20 jet engine sensors with remaining useful life labels. Public benchmark with reproducible metrics.

Inference pipeline

Training (normal data only):

minθ E [‖x − AEθ(x)‖²]

Anomaly threshold:

τ = percentile99(errorsval)

Production decision:

anomaly = (‖x − AEθ(x)‖² > τ)
